# Contributing

Thank you for your interest in contributing to **BanglaPDF2Markdown**!
🎉

Contributions of all kinds are welcome, including bug reports, feature
requests, documentation improvements, and code contributions.

## Getting Started

### 1. Fork the Repository

Click the **Fork** button on GitHub and clone your fork.

    git clone https://github.com/hello-hmemon/BanglaPDF2Markdown.git

    cd BanglaPDF2Markdown

### 2. Create a Virtual Environment

    python -m venv .venv

Windows:

    .venv\Scripts\activate

### 3. Install Dependencies

    pip install -r requirements.txt

## Running the Project

    python app.py sample.pdf

Or provide the full path to any PDF:

    python app.py "D:\Documents\sample.pdf"

## Running Tests

Run all tests using:

    pytest

## Coding Guidelines

Please follow these guidelines when contributing:

-   Keep the code simple and readable.
-   Follow the existing project structure.
-   Add comments only where necessary.
-   Write meaningful commit messages.
-   Test your changes before submitting a Pull Request.
-   Update documentation if your changes affect user-facing behavior.

## Pull Requests

Before opening a Pull Request, please ensure that:

-   Your code runs successfully.
-   Existing functionality is not broken.
-   New functionality is documented where appropriate.
-   Tests pass successfully.

## Reporting Bugs

When reporting a bug, please include:

-   Python version
-   Operating system
-   PDF sample (if possible)
-   Steps to reproduce the issue
-   Expected behavior
-   Actual behavior

## Suggesting Features

Feature requests are welcome.

Please explain:

-   What problem it solves
-   Why it would be useful
-   Any implementation ideas (optional)

## Questions

If you have questions, feel free to open a GitHub Issue.

Thank you for helping improve **BanglaPDF2Markdown**! ❤️
